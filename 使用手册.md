# 戏曲多模态 AI 传习平台 — 使用手册 v2.0

---

## 一、软件介绍

戏曲多模态 AI 传习平台是一款**淮剧智能化教学工具**，通过摄像头与麦克风实时分析用户的身段动作与唱腔发音，对比大师示范进行打分指导。支持音频/视频/图像多模态文件导入分析，生成专业特征图谱。

### 核心功能

| 功能 | 说明 |
|------|------|
| 🎭 传习工坊 | 摄像头实时姿态捕捉 + 声腔分析 + 大师对比 + 五级递进 |
| 🔮 知识图谱 | 3D 交互式戏曲知识网络（26 节点 33 关系） |
| 📜 剧本工坊 | 三级菜单浏览剧本，场次唱词全文 + 参考资源 |
| 🔬 识别分析 | 上传音频/视频/图像文件，自动提取特征生成分析报告 |
| 📊 学艺记录 | 历史训练会话列表，查看每次训练评分 |
| 📐 分析图表 | 系统架构图、姿态热力图、声腔频谱、能力雷达图 |

## 二、安装与启动

### 方式一：源码运行（推荐，需 Python 3.12+）

```bash
cd opera-ai
pip install -r requirements.txt

# 后端（端口 8090）
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8090

# 前端（另开终端）
cd frontend
npm install
npm run dev
```

浏览器访问 `http://localhost:5173`

### 方式二：打包 exe

```bash
python -m PyInstaller --onefile --name "opera-ai" --add-data "frontend/dist;frontend/dist" --add-data "graph;graph" --collect-all backend --hidden-import "uvicorn" --noconfirm launcher.py
```

> 注意：exe 打包约 200MB，启动需 10-20 秒。首次运行浏览器可能不会自动打开，手动访问 `http://127.0.0.1:8090`。

### 方式三：仅使用知识图谱 HTML

直接双击 `output/graph/opera-knowledge-graph.html`（独立页面，无需环境）。

## 三、传习工坊（核心功能）

### 3.1 选择练习剧本

开始训练前，**必须先选择剧本**：

1. 点击右上角红色「选择练唱剧本 ▲」按钮
2. 在弹出的三列面板中依次选择：
   - **剧种**（淮剧 / 京剧 / 豫剧）
   - **剧目**（女审 / 秦香莲 / 穆桂英挂帅）
   - **场次**（第一幕 / 第二幕 / 第三幕）
3. 选择完成后按钮变为绿色，显示所选剧本路径

### 3.2 开始识别

1. 浏览器弹出摄像头权限请求 → **点击"允许"**
2. 点击红色「开始识别」按钮
3. 系统开始实时分析你的动作和声音

### 3.3 界面布局

```
┌─────────────────┬─────────────────┐
│  摄像头画面      │  声腔波形分析    │
│  (你的实时影像)  │  (金色波形)      │
│  + 红色骨架叠加  │                 │
├─────────────────┼─────────────────┤
│  骨架对比       │  综合评分       │
│  左侧: 你的动作  │  评分环 + 分数   │
│  右侧: 大师示范  │  身段分 唱腔分   │
│                 │  纠偏建议       │
│                 │  进度条 + 导航   │
└─────────────────┴─────────────────┘
```

### 3.4 大师动作递进系统

系统内置《女审》第一幕 12 帧完整身段序列，分为五个等级：

| 等级 | 阈值 | 说明 |
|------|------|------|
| 初学 | 60 分 | 动作基本正确即可过关 |
| 熟练 | 70 分 | 动作流畅连贯，有戏曲韵味 |
| 精通 | 80 分 | 身段准确，水袖与台步配合得当 |
| 大师 | 90 分 | 收放自如，表情身段融为一体 |
| 完美 | 99 分 | 炉火纯青，达到舞台演出标准 |

**递进规则**：你的综合分达到当前帧的阈值后，连续保持 2 次即自动进入下一动作。右侧大师骨架会同步更新为新的示范动作。

### 3.5 手动控制

- **开始识别** / **暂停** / **继续** / **停止** — 控制训练流程
- **◀** — 回退到上一帧大师动作
- **起** — 重置到第一帧
- **▶** — 前进到下一帧

## 四、知识图谱

### 4.1 浏览图谱

1. 点击左侧导航「🔮 图谱」
2. 3D 球面分布显示 26 个戏曲知识节点
3. 鼠标拖拽旋转视角，滚轮缩放

### 4.2 查看节点详情

- **点击任意发光球体** → 弹出山水画卷详情页
- 画卷包含：
  - 节点名称、分类、属性（艺谱）
  - 所有关联节点卡片（艺缘）
  - 每个关联节点的关系标签、分类、属性摘要

### 4.3 搜索节点

1. 在顶部搜索框输入关键词（如"包公"）
2. 点击「搜索」或按 Enter
3. 结果自动聚焦，显示匹配节点及其邻居

### 4.4 关闭画卷

- 点击右上角 ✕ 按钮
- 或点击画卷外的暗色背景区域

## 六、识别分析（文件导入）

### 6.1 上传文件

1. 点击左侧导航「🔬 识别分析」
2. 点击上方按钮上传文件：
   - **🎵 音频**（WAV/MP3/FLAC/OGG/M4A）
   - **🎬 视频**（MP4/AVI/MOV/MKV/WEBM）
   - **🖼️ 图像**（JPG/PNG/BMP/WEBP）

### 6.2 分析功能

| 模态 | 分析内容 |
|------|----------|
| 音频 | MFCC 频谱图、F0 基频曲线、频谱质心、色度特征、能量包络 + 语谱图 + 唱句切分、波形图、滑音/颤音/润腔参数 |
| 视频 | MediaPipe 33 关节骨架提取、关节角度序列、运动学参数（角速度、可见度） |
| 图像 | Canny 边缘检测、轮廓提取、HSV 色彩直方图、边缘热力图对比 |

### 6.3 管理分析结果

- **右键项目** → 下载分析报告 / 复制下载链接 / 选择 / 删除
- **复选框多选** → 批量下载或批量删除
- **全选按钮** → 一键选中/取消全部
- **📥 ZIP 按钮** → 快速下载单个分析报告

### 6.4 分析报告内容

每个 ZIP 报告包含：
- `analysis.json` — 完整分析数据（特征参数、戏曲参数、错误信息）
- `mfcc.png`、`f0.png`、`centroid.png`、`chroma.png`、`energy.png`、`waveform.png` — 音频特征图谱
- 或 `angles.png`、`skeleton.png` — 视频骨架分析图
- 或 `canny.png`、`contours.png`、`hsv.png`、`comparison.png` — 图像分析图

### 6.5 格式兼容

- 自动嗅探文件头识别真实格式（MP4 伪装成 MP3 也能检测）
- 音频分析自动使用 ffmpeg 解码（限 10 分钟）
- 无 ffmpeg 时使用 librosa 回退解码

## 十、常见问题

**Q: 摄像头画面是黑的？**
A: 浏览器需要授权摄像头权限。检查地址栏左侧的摄像头图标，确保已允许。

**Q: 评分一直不动？**
A: 评分需要你实际活动身体，摄像头捕捉到运动后分数才会变化。尝试挥动手臂或走动。

**Q: 图谱搜索没结果？**
A: 尝试搜索"包公""淮调""水袖"等关键词。系统内置了 26 个节点回退数据。

**Q: 图表显示不出来？**
A: 点击左侧图表类型卡片后稍等 1-2 秒，后端会生成图表返回。

**Q: 音频上传后分析失败？**
A: 确认文件格式正确。系统支持 WAV/MP3/FLAC/OGG/M4A。如果显示"格式错误"，可能是 MP4 容器伪装成 MP3 后缀，请用视频导入。

**Q: ZIP 下载打不开/解压失败？**
A: 刷新页面后重试。如果问题持续，检查浏览器是否拦截了下载。

**Q: 端口被占用？**
A: 后端运行在 8090 端口，前端在 5173。关闭占用端口的进程后重试。

## 十一、技术架构

本项目遵循 **"采集—解析—关联—存储—展示—验证"** 六步递进架构：

1. **采集层**：多机位高清视频、高保真音频、戏服纹样扫描、剧本数字化
2. **解析层**：音频（MFCC/F0/频谱质心/色度）、视频（MediaPipe 骨架）、图像（Canny 边缘/YOLOv8 纹样）
3. **关联层**：统一实体 ID 映射，剧目—唱段—艺人三级主键关联
4. **存储层**：Neo4j 图数据库 + 文件系统（分析结果落盘）
5. **展示层**：React + Three.js 3D 前端 + FastAPI 后端
6. **验证层**：唱腔相似度评分、身段骨架比对（AI 传习）
